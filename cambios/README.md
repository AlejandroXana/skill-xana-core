# Cambios

Features, refactors, migraciones relevantes aplicados a la plataforma Xana Core.

**Formato de archivo**: `YYYY-MM-DD-slug.md` (cronológico).

**Contenido esperado en cada archivo**:
- Qué se cambió
- Por qué (motivación, trigger, incidente relacionado)
- Alcance: qué sitios/repos/componentes
- Detalles técnicos relevantes
- Pendiente / siguientes pasos (si procede)

No para bugs resueltos — eso va en `incidencias/`. No para decisiones con trade-offs documentados — eso va en `decisiones/`.
